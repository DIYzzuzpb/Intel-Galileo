耳机接口, 从上至下, 分别是 左声道, 右声道, 地。
耳机左声道(紫色)接TTL的GND
耳机右声道(绿色)接TTL的TXD
耳机地    (红色)接TTL的RXD

上面的接法莫名有bug,只有输出没有输入。。。会不会因为USB转TTL的那个DTR引脚悬空呢?
debug: Galileo板子上的耳机接口, 输出的RS232电平!!! 不要什么串口调试都以为是TTL哎~~~错误!

那么为什么有输出没有输入呢? 即RS232的电平, 可以被TTL转USB模块识别并正常输出, 
而要输入的数据, 听过USB转TTL后, 得到的是TTL的电平, RS232不接受?

TTL电平:
输出 L： <0.8V ； H：>2.4V。
输入 L： <1.2V ； H：>2.0V

RS232电平:
逻辑1的电平为-3～-15V，逻辑0的电平为+3～+15V，注意电平的定义反相了一次。

九针串口接法

耳机左声道接 九针串口的2 input RxD
右声道接     九针串口的3 output Txd
地接         九针串口的5 ground

             九针串口的4 output DSR 与 6 input DSR连接
             九针串口的7 output RTS 与 8 input RTS连接
